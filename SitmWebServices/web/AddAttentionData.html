AddAttentionData

Adds attention data to the database

Params

  • appKey – the application key. Required.
  • srcKey – the key of the source of the attention (typically a user/listener). Required
  • tgtKey – the key of the target of the attention. Required.
  • type – the type of the attention. Required. May be one of:
    • DISLIKED – the source doesn't like the target
    • PLAYED – the source played the target
    • LOVED – the source loves the target
    • RATING – the source rated the target
    • TAG – the source tagged the target
  • value – the context-dependent value associated with the attention. Required for attention types RATING and TAG, optional for attention type PLAYED
    • RATING value – a number between 1 (worst) and 5 (best)
    • PLAYED value – the play count. Default is 1.
    • TAG value – the string tag.
Note Perhaps the DISLIKED and LOVED can be replaced with RATING

Examples

Request

    http://music.tastekeeper.com/api/AddAttentionData
        ?appKey=SAMPLE_APP_ID
        &srcKey=lamere
        &tgtKey=6fe07aa5-fec0-4eca-a456-f29bff451b04&type=PLAYED

Results

The format of the results depend upon the type of item being retreived 


<AddAttentionData>
    <request>
        <param name="tgtKey">6fe07aa5-fec0-4eca-a456-f29bff451b04</param>
        <param name="type">PLAYED</param>
        <param name="srcKey">lamere</param>
        <param name="appKey">SAMPLE_APP_ID</param>
    </request>
    <results status="OK">
        <time ms="40"/>
    </results>
</AddAttentionData>
