Project Aura Web Service

The Project Aura music recommendation web service allows anyone to build a music service using Project Aura's unique transparent and steerable recommendations.

Overview

The API loosely follows REST-style conventions. For some of the methods, you will need to use an application that you can obtain from the here: [[TBD-application-key-registration-site]].

The API root URL is http://music.tastekeeper.com/api/1.0/

API methods are of the form http://music.tastekeeper.com/api/1.0/method?arg1=value1&arg2=value



Methods

  • AddListener

  • ArtistSearch – searches for an artist by name

  • ArtistTagSearch

  • FindSimilarArtist – finds artists that are similar to a seed artist

  • FindSimilarArtistFromWordCloud

  • FindSimilarArtistTags

  • FindSimilarListener

  • GetApml

  • GetArtists – gets the most popular artists

  • GetArtistTags – Gets the Tags that have been applied to an artist

  • GetAttentionData

  • GetItem

  • GetListenerTags

  • GetRecommendations

  • GetStats

  • GetTags

  • TagItem

Encoding

Use UTF-8 encoding when sending arguments to the API methods.

Status

Status is returned as a <status> element that includes a status code and message. For example:

<FindSimilarArtist>
	<Status code="MissingArgument">this method needs a name or a key</Status>
</FindSimilarArtist>



Possible status codes are:



Status Code

Description

OK

No Error

InternalError

An internal error occurred

MissingArgument

A required argument is missing

BadArgument

An argument is in an improper format

NotFound

The requested item was not found

InvalidKey

The application key is not valid for the requested operation

RateLimitExceeded

The application has exceeded the allowed rate

InvalidMethod

The requested method is invalid

Offline

The webservice is currently offline



Scores

Many results have a score associated with them. Scores range from 0.0 to 1.0.

Popularity

Some results have popularity associated with them. Popularity ranges from 0.0 to 1.0.

FindSimilarArtist

Finds artists that are similar to a seed artist.

Params

  • name – the name of the artist. Wildcards are allowed. If multiple artists match the name, the best matching artist is used. Required if key not present.

  • key – the musicbrainz ID for the artist. Required if name not present

  • max – the maximum number of results returned. Optional, default is 10.

  • popularity – the desired range of popularity in the results. Optional, default is 'all'. Value must be one of:

    • all – all artists are considered

    • head – only very popular artists are considered

    • mid – only moderately popular artists are considered

    • tail – only long tail artists are considered

    • head_mid – very and moderately popular artists are considered

    • mid_tail – very popular artists are not considered.

  • field - the field to use for similarity. Optional, default is 'socialTtags'. Value must be one of:

    • all – consider all fields for similarity

    • autoTags – consider autotags for similarity

    • biographySummary – use the artist biography for similarity

    • collaborations – use the artist collaborations for similarity

    • events – use the artist

    • followers – use the artist followers for similarity

    • influencers – use the aritst influencers for similarity

    • relatedArtists – use the related artists for similarity

    • socialTags- use the social tags for similarty

    • bioTags – use bio tags (the artist biography filtered for music-related terms)

    • blurbTags – use blurb tags (artist reviews filtered for music-related terms)

For the future – we will allow specification of multiple weighed fields for similarity like so: field=(autoTags,1)(socialTags,2)(blurbTags,.5)

Example

http://www.tastekeeper.com/api/1.0/FindSimilarArtist?name=weezer&popularity=head

GetArtistTags

Gets the tags that have been applied to an artist

Params

  • name – the name of the artist. Wildcards are allowed. If multiple artists match the name, the best matching artist is used. Required if key not present.

  • key – the musicbrainz ID for the artist. Required if name not present

  • max – the maximum number of results returned. Optional, default is 100.

  • type – the type of tag report – distinctive or frequent. Optional, default is distinctive. When type is distinctive, the highest weighted tags are returned, otherwise, the tags with the highest frequency are returned.

  • field - the field of interest. Optional, default is 'socialTtags'. Value must be one of: all, autoTags, biographySummary, collaborations, events, followers, influencers, relatedArtists, socialTags, bioTags, blurbTags

Example

http://www.tastekeeper.com/api/1.0/GetArtistTags?max=10&name=weezer
